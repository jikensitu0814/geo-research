# Support

## What To Open

Use GitHub Issues for:

- Data error reports in public aggregate tables.
- Methodology questions or suggested clarifications.
- Benchmark proposals for new industries, prompt sets, model interfaces, or metrics.
- Validator bug reports.
- Documentation fixes.

## What Not To Open

Please do not request:

- Raw AI response exports.
- SQLite databases.
- Client-facing reports.
- Private run logs or local execution traces.
- Credentials, environment files, or unpublished URL inventories.

## Response Expectations

This is a maintained public research package, not a real-time support channel. Data-safety and validator issues are prioritized first, followed by methodology corrections and benchmark proposals.
